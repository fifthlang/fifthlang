# Requirements Document

## Introduction

This feature adds a fully working sample solution at `samples/FullProjectExample/` that demonstrates multi-project Fifth language development at scale using MSBuild. The sample uses the new SLNX solution format (`.slnx`) and includes a console application with class library dependencies wired through `ProjectReference`. An accompanying Getting Started guide in `docs/Getting-Started/` explains how to reproduce the setup in Visual Studio.

## Glossary

- **SLNX_Solution**: An XML-based solution file (`.slnx` format) introduced in .NET 9 / Visual Studio 17.10+ that replaces the legacy `.sln` text format with clean, human-readable XML.
- **Fifth_Project**: A `.5thproj` MSBuild project file that uses `<Project Sdk="Fifth.Sdk">` to build Fifth language source files (`.5th`).
- **Console_App**: A Fifth_Project with `<OutputType>Exe</OutputType>` that serves as the executable entry point of the solution.
- **Class_Library**: A Fifth_Project with `<OutputType>Library</OutputType>` that produces a `.dll` and is consumed by other projects via ProjectReference.
- **Fifth_Sdk**: The custom MSBuild SDK located at `src/Fifth.Sdk/` (version 0.1.0) that provides build targets for `.5thproj` files.
- **NuGet_Config**: A `NuGet.Config` file that configures package sources, including the local Fifth_Sdk package source for SDK resolution.
- **Global_Json**: A `global.json` file that pins the Fifth_Sdk version via the `msbuild-sdks` section.
- **Getting_Started_Guide**: A Markdown document in `docs/Getting-Started/` that explains how to set up and build the sample solution.
- **ProjectReference**: An MSBuild element that declares a dependency from one Fifth_Project to another, enabling the `ResolveFifthProjectReferences` target to build dependencies in order.
- **Source_Manifest**: A file generated by MSBuild listing all `.5th` source files for namespace-aware multi-file compilation.

## Requirements

### Requirement 1: SLNX Solution File

**User Story:** As a Fifth developer, I want a sample SLNX solution file, so that I can see how to organize multiple Fifth projects in the modern XML-based solution format.

#### Acceptance Criteria

1. THE SLNX_Solution SHALL be located at `samples/FullProjectExample/FullProjectExample.slnx`.
2. THE SLNX_Solution SHALL use the `<Solution>` root element with `<Project>` child elements referencing each Fifth_Project by relative path.
3. THE SLNX_Solution SHALL include the Console_App project and all Class_Library projects.
4. WHEN opened in Visual Studio 17.10+ or built with `dotnet build`, THE SLNX_Solution SHALL resolve all referenced Fifth_Project files without errors.

### Requirement 2: Console Application Project

**User Story:** As a Fifth developer, I want a console application project as the solution entry point, so that I can see how to structure an executable Fifth project that depends on class libraries.

#### Acceptance Criteria

1. THE Console_App SHALL be located at `samples/FullProjectExample/src/App/App.5thproj`.
2. THE Console_App SHALL set `<OutputType>Exe</OutputType>` and `<TargetFramework>net8.0</TargetFramework>`.
3. THE Console_App SHALL declare `<ProjectReference>` elements pointing to each Class_Library project.
4. THE Console_App SHALL contain at least one `.5th` source file with a `main(): void` entry point.
5. WHEN built, THE Console_App SHALL produce an executable at `bin/<Configuration>/net8.0/App.exe`.

### Requirement 3: Class Library Projects

**User Story:** As a Fifth developer, I want class library projects in the sample, so that I can see how to structure reusable Fifth libraries that are consumed via ProjectReference.

#### Acceptance Criteria

1. THE sample SHALL include at least two Class_Library projects located under `samples/FullProjectExample/src/`.
2. Each Class_Library SHALL set `<OutputType>Library</OutputType>` and `<TargetFramework>net8.0</TargetFramework>`.
3. Each Class_Library SHALL contain at least one `.5th` source file with function definitions.
4. WHEN built, each Class_Library SHALL produce a `.dll` at `bin/<Configuration>/net8.0/`.

### Requirement 4: ProjectReference Wiring

**User Story:** As a Fifth developer, I want to see proper ProjectReference wiring between projects, so that I can understand how multi-project dependency resolution works with Fifth_Sdk.

#### Acceptance Criteria

1. THE Console_App SHALL declare `<ProjectReference>` elements for each Class_Library it depends on.
2. WHEN the Console_App is built, THE Fifth_Sdk `ResolveFifthProjectReferences` target SHALL build all referenced Class_Library projects before compiling the Console_App.
3. One Class_Library MAY declare a ProjectReference to another Class_Library to demonstrate transitive dependency chains.

### Requirement 5: SDK Resolution Configuration

**User Story:** As a Fifth developer, I want the sample to include local SDK resolution configuration, so that I can build the sample without publishing Fifth_Sdk to a remote NuGet feed.

#### Acceptance Criteria

1. THE sample SHALL include a `NuGet.Config` file at `samples/FullProjectExample/NuGet.Config`.
2. THE NuGet_Config SHALL define a `local-sdk` package source pointing to the relative path of the Fifth_Sdk build output (`../../src/Fifth.Sdk/bin/Debug`).
3. THE NuGet_Config SHALL use `packageSourceMapping` to route the `Fifth.Sdk` package pattern to the `local-sdk` source.
4. THE sample SHALL include a `global.json` file at `samples/FullProjectExample/global.json`.
5. THE Global_Json SHALL specify `"Fifth.Sdk": "0.1.0"` in the `msbuild-sdks` section.

### Requirement 6: Fifth Source Files

**User Story:** As a Fifth developer, I want minimal but valid Fifth source files in each project, so that the sample compiles and demonstrates the build pipeline without requiring deep language knowledge.

#### Acceptance Criteria

1. Each `.5th` source file SHALL contain syntactically valid Fifth code that the compiler accepts.
2. THE Console_App source file SHALL define a `main(): void` function as the entry point.
3. Each Class_Library source file SHALL define at least one function.
4. THE source files SHALL be kept minimal, focusing on demonstrating project structure rather than language features.

### Requirement 7: Build Output Hygiene

**User Story:** As a Fifth developer, I want a `.gitignore` in the sample directory, so that build outputs are excluded from version control.

#### Acceptance Criteria

1. THE sample SHALL include a `.gitignore` file at `samples/FullProjectExample/.gitignore`.
2. THE `.gitignore` SHALL exclude `bin/` and `obj/` directories.
3. THE `.gitignore` SHALL exclude common IDE artifacts (`.vs/`, `*.user`).

### Requirement 8: Getting Started Documentation

**User Story:** As a Fifth developer, I want a step-by-step guide explaining how to set up a multi-project Fifth solution in Visual Studio, so that I can replicate the sample structure for my own projects.

#### Acceptance Criteria

1. THE Getting_Started_Guide SHALL be located at `docs/Getting-Started/full-project-setup.md`.
2. THE Getting_Started_Guide SHALL explain what the SLNX format is and why it is used instead of the legacy `.sln` format.
3. THE Getting_Started_Guide SHALL describe step-by-step how to create the solution directory structure.
4. THE Getting_Started_Guide SHALL describe how to create each `.5thproj` file (Console_App and Class_Library) with the correct properties.
5. THE Getting_Started_Guide SHALL describe how to configure ProjectReference elements between projects.
6. THE Getting_Started_Guide SHALL describe how to set up `NuGet.Config` and `global.json` for local Fifth_Sdk resolution.
7. THE Getting_Started_Guide SHALL describe how to build and run the solution using `dotnet build` and `dotnet run`.
8. THE Getting_Started_Guide SHALL describe how the sample maps to Visual Studio workflows (opening the `.slnx`, building from the IDE, setting the startup project).
9. THE Getting_Started_Guide SHALL be added to the mkdocs navigation in `mkdocs.yml` under the Getting Started section.

### Requirement 9: FifthCompilerPath Configuration

**User Story:** As a Fifth developer, I want each project to specify the FifthCompilerPath, so that the sample builds correctly from the repository without relying on auto-resolution from the SDK package layout.

#### Acceptance Criteria

1. Each Fifth_Project SHALL set `<FifthCompilerPath>` to the relative path of the compiler DLL within the repository (`../../../../src/compiler/bin/Release/net8.0/compiler.dll` or equivalent relative path from each project directory).
2. IF the compiler DLL is not found at the specified FifthCompilerPath, THEN THE Fifth_Sdk SHALL report a clear error message indicating the compiler must be built first.
